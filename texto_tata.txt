Não interpretar aspas não fechadas ou caracteres especiais que não são 
necessários para o assunto, como \ (barra invertida) ou ; (ponto e vírgula).
Lidar com ' (apóstrofo) que deve impedir o shell de interpretar os 
metacaracteres na sequência citada.
Lidar com " (aspas duplas) que deve impedir o shell de interpretar os 
metacaracteres na sequência citada, exceto por $ (cifrão).
Implementar redirecionamentos:

    < deve redirecionar a entrada.

     >   deve redirecionar a saída.

    << deve receber um delimitador e, em seguida, ler a entrada até que uma linha contendo o delimitador seja vista. No entanto, não precisa atualizar o histórico!

          >>  deve redirecionar a saída no modo de anexação.

Implementar tubulações (caractere |). A saída de cada comando na tubulação
 está conectada à entrada do próximo comando por meio de um tubo.
Lidar com variáveis de ambiente ($ seguido de uma sequência de caracteres) 
que devem se expandir para seus valores.
Lidar com $? que deve se expandir para o status de saída do pipeline 
mais recentemente executado em primeiro plano.
